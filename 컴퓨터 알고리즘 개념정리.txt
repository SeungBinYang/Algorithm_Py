ex) 정수 int 실수 float

ex) 선형리스트, 연결리스트, 스택, 큐

비선형 자료구조 : 하나의 데이터 뒤에 여러 데이터가 연결되는 구조

알고리즘 표현 : 자연어 코드 , ... 


시간복잡도 (빅오표기법)

 = 1차원 리스트 여러개를 묶은 구조.

3. 딕셔너리
 = 키:값 형태로 저장하는 자료형.
 = { } 사용함
 = 키가 유일해야 함.
 = 순서 중심이 아니라 키 중심 접근이다.
 = dict[키], dict.get(키) 로 값에 접근

리스트는 위치 번호로 접근하고, 딕셔너리는 키로 접근한다.

4. 세트
 = 중복 없는 값들의 집합
 = 중괄호를 사용하지만 딕셔너리처럼 : 가 없다.
 = 같은 값이 여러번 들어가도 하나만 남고, 교집합, 합집합, 차집합 연산이 가능하다.

5. 문자열

 = 검색, 분리, 결합 등 여러 함수가 있는데, 예를 들어 find(), split(), join() 등이 있다.

6. 튜플
 = 리스트와 비슷하지만 수정할 수 없는 자료형
 = ( )를 사용하고, 읽기 전용 데이터에 적합한 구조
 = 항목이 하나인 튜플은 반드시 쉼표가 필요하다.

리스트와 튜플 비교.
 - 리스트는 수정 가능.
 - 튜플은 수정 불가.

선형 리스트
 = 데이터를 일정한 순서로 나열한 자료 구조.
 = 줄서기처럼 순서대로 나열된 구조. 일정한

 = 순차 리스트 라고 하고, 보통 배열 기반으로 구현함

즉, 선형 리스트는 메모리상에서 데이터가 논리적으로 순서 있게 나열된 구조.

장점 : 위치에 바로 접근할 수 있음.
	배열의 0번, 1번, 2번 처럼 인덱스로 접근하므로 원하는 위치에 데이터를 빠르게 찾을 수 있다.

단점 : 중간에 데이터를 삽입하거나 삭제할 때 시간이 많이 소요됨.
	ex) 중간에 하나를 넣기 위해선, 뒤에 있는 원소를 전부 한칸씩 뒤로 밀어야 했고,
	     삭제 시에도, 뒤에 있는 값들을 앞으로 당겨와야 하였음.

	    => 삽입과 삭제의 오버헤드가 크다.

선형 리스트의 삽입 원리
	1. 빈칸은 추가하고,
	2. 2번까지 마지막 데이터에서 한칸 뒤로 미룸.
	3. 2번 위치에 값을 수정.

선형 리스트의 삭제 원리
	1. 삭제할 위치에 데이터를 제거
	2. 그 뒤의 데이터를 한 칸씩 앞으로 당기고
	3. 마지막 남는 칸을 제거.

	=> 삭제도 많은 이동이 필요하다.

선형 리스트는 접근이 빠르지만. 중간 삽입/삭제가 느리다.

선형 리스트의 응용
	= 다항식을 선형 리스트로 표현한 예
	= 최고차항부터 계수를 차례로 저장해서 계산하거나 출력하는 방식.
	= 특수 다항식처럼 0이 아닌 항만 저장하는 예.
	= 선형 리스트가 단순한 데이터 저장을 넘어서 실제 문제 표현에 쓰일 수 있다.

단순 연결 리스트
	= (노드) 들이 물리적으로 떨어진 위치에 존재하고, 각 (노드)가 다음 (노드)를 가리키는 링크를 통해 연결된 구조
	= 즉, 배열처럼 연속된 메모리에 저장되는게 아니고, 여기저기 흩어져 있는 노드들을 화살표로 이어놓은 형태

노드의 구조
	= 단순 연결 리스트의 기본 단위는 노드
	= 노드는 데이터, 링크(= 다음 노드를 가리키는 주소)로 이뤄짐.
	
즉, 배열처럼 값만 저장하는게 아니라, 다음 노드 정보까지 함께 저장해야 함.

장점
1. 삽입/삭제가 유리함.
2. 배열처럼 원소들을 이동할 필요없이, 앞 뒤 링크만 수정하면 되기 때문

단점
1. 원하는 위치에 바로 갈 수 없다.(데이터 접근이 느리고, 탐색 방향이 단일이라는 한계)
2. 메모리 낭비

기본 변수
3개 (head, current, pre)

단순 연결 리스트의 생성
1. 처음에 비어 있는 상태에서 시작한다.
2. 데이터를 넣을 때 head 가 그 노드를 가리키게 하면 된다.
3. 새 노드를 만들어 마지막 노드의 링크에 연결한다.

즉, 처음 노드와 그 이후 노드의 처리 방식이 다르므로, 보통 조건문이 들어간다.

노드 검색
= 검색은 head 부터 시작 ... 현재 노드의 데이터가 찾는 값과 같은지 비교하고, 아니면 다음 노드로 이동함.
= 끝까지 못 찾으면 None을 반환함.

배열처럼 바로 특정 위치를 읽는 것이 아니라, 항상 앞에서부터 순차적으로 찾아가야한다는 점이 연결 리스트의 특징

연결 리스트의 응용
= 이름과 연락처를 무작위로 입력받아 이름 순으로 정렬되도록 연결 리스트
= 연결 리스트가 단순히 저장만 하는 구조가 아니라, 삽입하면서 순서를 유지하는 구조로도 활용될 수 있다.
= 새 데이터가 첫 노드보다 작으면 맨 앞, 가장 크면 맨 뒤에 삽입.

선형 리스트와 단순 연결 리스트

1. 저장 방식
	- 선형 리스트 : 배열처럼 순차적으로 저장
	- 단순 연결 리스트 : 노드가 흩어져 있고 링크로 연결 

2. 접근 방식
	- 선형 리스트 : 인덱스로 즉시 접근 가능
	- 단순 연결 리스트 : head 부터 순차적으로 따라가야 함.

	접근 속도는 선형 리스트가 유리
	
3. 삽입 / 삭제
	- 선형 리스트 : 데이터의 이동이 많아 비효율적임.
	- 단순 연결 리스트 : 링크만 수정하면 되서 효율적임.
